# WebSearch Query Patterns

Technology-specific query templates for effective documentation research.

## Frontend Frameworks

### React
```
react {version} {topic} official documentation
react {topic} hooks example
react {component-pattern} best practices
react {error-message} solution stackoverflow
```

### Next.js
```
nextjs {version} {topic} documentation
nextjs app router {topic}
nextjs {feature} tutorial vercel
```

### Vue
```
vue 3 {topic} composition api
vue {topic} official guide
vuejs {pattern} example
```

## Backend & Runtime

### Node.js
```
nodejs {version} {module} documentation
node {topic} best practices
nodejs {error} solution
```

### Python
```
python {version} {module} documentation
python {topic} official docs
{library} python tutorial
```

### Effect (TypeScript)
```
effect-ts {topic} documentation
effect typescript {pattern}
effect.website {topic}
```

## Databases

### Prisma
```
prisma {topic} documentation
prisma {relation-type} example
prisma {error} solution github
```

### PostgreSQL
```
postgresql {version} {topic} documentation
postgres {feature} example
psql {command} usage
```

## APIs & Auth

### NextAuth.js / Auth.js
```
nextauth {version} {provider} setup
auth.js {topic} documentation
nextauth {session-strategy} configuration
```

### REST/GraphQL
```
{api-name} api documentation
{service} api {endpoint} example
graphql {pattern} best practices
```

## DevOps & Tools

### Docker

```
dockerfile {language} best practices

docker {error} solution
```

### Git
```
git {command} documentation
git {workflow} best practices
github actions {task} example
```

## Query Modifiers

Add these to narrow results:

| Modifier | Use Case |
|----------|----------|
| `site:github.com` | Official examples |
| `site:stackoverflow.com` | Problem solutions |
| `after:2024` | Recent content only |
| `"{exact phrase}"` | Specific error messages |
| `-site:medium.com` | Exclude low-quality sources |

## Version-Specific Queries

When version matters:

```
{technology} {version} breaking changes
{technology} {version} migration guide
{technology} {version} {topic} (not {old-version})
```

## Error-Based Queries

For debugging:

```
"{exact error message}" {technology}
{technology} {error-code} cause
{technology} {error-type} fix 2024
```
